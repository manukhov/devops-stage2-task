# Тестовое задание — этап 2

Нужно доработать инфраструктурный проект на Docker Compose.

## Задача
Собрать воспроизводимое решение, которое включает:

- тестовое приложение;
- PostgreSQL;
- Prometheus;
- Grafana;
- базовый backup/restore для PostgreSQL;
- CI pipeline.

## Что нужно сделать

1. Доработать проект так, чтобы он запускался через Docker Compose.
2. Обеспечить работу приложения с PostgreSQL.
3. Настроить Prometheus для сбора метрик.
4. Настроить Grafana так, чтобы datasource и dashboard поднимались автоматически из репозитория.
5. Добавить базовый механизм backup/restore PostgreSQL.
6. Добавить CI pipeline с базовой проверкой проекта.
7. Добавить README:
   - как запустить;
   - как проверить;
   - как сделать backup;
   - как сделать restore;
   - какие есть ограничения решения;
   - как бы вы развивали его дальше для production / HA.

## Что мы оцениваем
- работоспособность;
- воспроизводимость;
- observability;
- backup/restore;
- CI;
- качество README;
- инженерную логику.

## Формат результата
Ссылка на Git-репозиторий или архив с проектом.
